COINS = [100, 99, 66, 33, 9, 6, 3, 1]

Я задав незвичний номінал монет щоб продемонструвати що жадібний алгоритм не завжди знаходить оптимальне рішення, особливо для наборів монет, де не можна застосувати жадібну стратегію для отримання мінімальної кількості монет. Алгоритм динамічного програмування, незважаючи на його повільніше виконання, забезпечує точні та оптимальні результати.

Часова складність жадібного алгоритму - O(N)
Часова складність цього алгоритму є O(K×S), де K - кількість номіналів монет, а S - сума.

Час на жадібний алгоритм: 3.275007475167513e-05
